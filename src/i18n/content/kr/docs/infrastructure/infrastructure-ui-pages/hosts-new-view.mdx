**Hosts** \[호스트] 페이지와 해당 타일을 사용하여 리소스 사용량, 네트워크 및 프로세스 성능, 컨테이너에 대한 데이터 및 서로 간의 관계와 같은 중요한 성능 지표를 더 잘 이해할 수 있습니다. 속성별로 호스트를 필터링하고 성능을 인벤토리 변경 이벤트와 비교하여 근본 원인을 파악하는 데 도움을 받을 수 있습니다. 문제를 방지하는 데 도움이 되도록 경고 알림을 설정할 수도 있습니다.

## 새 인프라 UI에 액세스 [#access-new-ui]

호스트 정보를 보려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > Hosts**로 이동하고 **Overview** \[개요] 탭을 클릭하십시오.

새 UI에는 상위 수준에서 호스트의 진행 상황을 확인하는 데 도움이 되는 요약 타일이 포함되어 있으며, 신속하게 드릴다운하여 특정 호스트 세부 정보를 보고 해당 호스트별로 필터링할 수 있습니다.

## 필터 및 저장된 보기 [#filter]

페이지 상단의 필터를 사용하여 제안된 필터를 사용하거나 연결된 이름 또는 태그를 입력하여 호스트를 찾으십시오.

인프라에 대해 저장된 보기가 있는 경우 **Saved views**\[저장된 보기] 버튼을 통해 액세스할 수 있습니다.

## 호스트 탭 [#tabs]

**호스트** 페이지에서 사용할 수 있는 탭은 다음과 같습니다.

<CollapserGroup>
  <Collapser
    id="summary"
    title="요약"
  >
    **Summary** 탭은 호스트 성능에 대한 개요를 제공합니다.

    페이지 상단에 있는 4개의 타일은 호스트와 해당 애플리케이션, 이벤트 및 경고에 대한 높은 수준의 개요를 제공합니다. 이러한 타일을 사용하여 호스트에 대한 자세한 내용을 보거나 특정 호스트별로 전체 요약 보기를 필터링할 수 있습니다.

    ### 호스트

    호스트 타일에는 계정의 총 호스트 수가 표시됩니다. 모든 호스트 목록을 보려면 클릭하십시오. 해당 목록에서 특정 호스트를 검색하거나 찾습니다. 특정 호스트 필터를 해당 호스트로 확장하거나 더 많은 데이터를 표시합니다.

    <img
      title="Host metadata"
      alt="Host metadata"

      style={{ height: '100%', width: '100%'}}
    />

    <figcaption>
      **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > Hosts (New)**에서 **Hosts** \[호스트] 타일을 클릭하고 호스트를 선택합니다.
    </figcaption>

    해당 호스트에 대한 자세한 메타데이터를 보려면 **Show more** \[더 보기] 버튼을 클릭하십시오.

    **Filter to host** \[호스트로 필터링] 버튼을 클릭하여 **Hosts** \[호스트] 페이지를 해당 호스트로 필터링합니다. 전체 보기는 해당 호스트의 애플리케이션, 이벤트, 경고 및 페이지의 다른 모든 차트를 필터링합니다.

    ### 애플리케이션

    <img
      title="Applications tile plus list"
      alt="applications tile plus list"
      src={infrastructureHostApplication}
      style={{ height: '100%', width: '100%'}}
    />

    <figcaption>
      **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > Hosts (New)** 에서 **Applications** \[애플리케이션] 타일을 클릭하고 애플리케이션을 선택합니다.
    </figcaption>

    애플리케이션 타일은 호스트에 직접 설치된 총 APM 애플리케이션 수를 보여줍니다. 이 숫자는 적용한 호스트 필터에 따라 변경됩니다.

    필터 단추를 클릭하면 해당 응용 프로그램에 대한 관련 호스트, 이벤트 및 알림은 물론 해당 응용 프로그램에 대해 필터링된 페이지의 다른 모든 차트도 볼 수 있습니다.

    해당 애플리케이션에 대한 APM 요약 페이지를 보려면 **View in APM** \[APM에서 보기]를 클릭합니다.

    ### 이벤트

    이벤트 타일에는 호스트와 관련된 총 이벤트 수가 표시됩니다.

    <img
      title="Infrastructure events page"
      alt="Infrastructure events page"
      src={infrastructureEventsPage}
      style={{ height: '100%', width: '100%'}}
    />

    <figcaption>
      **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > Hosts (New)** 에서 **Events** \[이벤트] 타일을 클릭하여 이벤트 페이지로 이동합니다.
    </figcaption>

    **Events** \[이벤트] 타일을 클릭하여 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > Events** 의 **Events** \[이벤트] 페이지로 이동합니다.

    ### 경고

    알림 타일은 위험 또는 경고 상태에 있는 호스트의 수를 보여줍니다. 타일을 클릭하여 자세한 내용을 확인한 다음 특정 인시던트를 클릭하여 기타 관련 문제, 근본 원인 분석 및 기타 영향을 받는 엔터티를 확인합니다.

    <img

    <figcaption>
      **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > Hosts (New)** 에서 **Alerting** \[경보] 타일을 클릭하고 경보 활동을 확인하십시오.
    </figcaption>
  </Collapser>
